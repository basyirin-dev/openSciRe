**Duration**: Post-pilot, before or during YC application
**Dependencies**: Phase 13 (pilots), Phase 14 (distribution)
**Output**: Public repository with community infrastructure

### Task 15.1: Repository Public Launch

- [ ] 15.1.1: Make repository public (if private)

- [ ] 15.1.4: Social media campaign (Twitter/X, LinkedIn, Mastodon)
- [ ] 15.1.5: First public release `v0.1.0` with changelog

### Task 15.2: Community Infrastructure

- [ ] 15.2.1: `CONTRIBUTING.md` — coding standards, PR workflow, ethics guidelines
- [ ] 15.2.2: `CODE_OF_CONDUCT.md` — adapted Contributor Covenant
- [ ] 15.2.3: GitHub Issue templates refined from Phase 0
- [ ] 15.2.4: GitHub Discussions enabled for Q&A, ideas, show-and-tell
- [ ] 15.2.5: Discord/Slack community server with channels: #general, #help, #showcase, #ethics, #development
- [ ] 15.2.6: Community moderation guidelines and moderator onboarding

### Task 15.3: Trusted Tester Program

- [ ] 15.3.1: Public call for Trusted Testers (PhD students, postdocs, PIs)
- [ ] 15.3.2: Application form — research domain, experience level, tools used, commitment
- [ ] 15.3.3: Onboarding flow — same as Phase 13.2 but self-service
- [ ] 15.3.4: Trusted Tester Slack channel — priority support, feature voting, early access

### Task 15.4: Adversarial Red Team Program

- [ ] 15.4.1: Public call for Red Team participants — security researchers, philosophers of science, ethicists, domain experts
- [ ] 15.4.2: Structured red teaming scenarios:
  - Try to generate DURC hypotheses (bypass ethical firewall)
  - Try to falsify provenance entries (tamper with signed records)
  - Try to inject false citations (bypass source grounding)
  - Try to hide carbon-intensive queries (bypass carbon tracker)
- [ ] 15.4.3: Bug bounty (if budget permits) or recognition program
- [ ] 15.4.4: Findings disclosure process — responsible disclosure timeline, public acknowledgment
- [ ] 15.4.5: Red team report published (anonymized) — builds trust, shows commitment to safety

### Task 15.5: Content & Outreach

- [ ] 15.5.1: Blog posts:
  - "The eight gaps in Gemini for Science" (link to critiques)
  - "What we learned building a falsification agent" (technical deep-dive)
  - "Provenance-native AI: Why your research tool needs a signed audit trail"
  - "Carbon-aware AI research: tracking the cost of a hypothesis"
- [ ] 15.5.2: Conference presence — submit demo/poster to:
  - NeurIPS workshops (Socially Responsible AI, ML for Science)
  - ISMB (International Symposium on Molecular Biology)
  - SACNAS (Society for Advancing Chicanos/Hispanics and Native Americans in Science)
  - SciPy / PyData
- [ ] 15.5.3: Demo video — 3-minute walkthrough of "research question → hypothesis → falsification → ethics review"
