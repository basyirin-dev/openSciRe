# Phase 13 — Academic Pilot Program

**Duration**: 12 weeks (Jan–Mar 2027, parallel with Phase 10)
**Dependencies**: Phase 8 (CLI), Phase 10 (Jupyter)
**Output**: 3–5 pilot labs with demonstrated traction for YC application

---

### Task 13.1: Lab Recruitment

- [ ] 13.1.1: Identify target labs from Phase 0.5 desk research segments and community engagement
- [ ] 13.1.2: Develop pilot value proposition — free access to managed version, priority feature requests, co-authorship on validation paper
- [ ] 13.1.3: Create pilot agreement — scope, timeline, data handling, publication rights, feedback expectations, NDA (if needed)
- [ ] 13.1.4: Recruit 3–5 labs (diverse: molecular biology, ecology, clinical, computational chemistry)
- [ ] 13.1.5: Ensure at least 2 labs have a concrete research question to start with

### Task 13.2: Onboarding

- [ ] 13.2.1: Pre-onboarding survey — current tools, workflow, pain points (baseline metrics)
- [ ] 13.2.2: Installation support — pip install, conda env, Ollama setup (if using local models)
- [ ] 13.2.3: First session walkthrough — guided "search → read → hypothesize" workflow
- [ ] 13.2.4: Customization — configure BYOK endpoints if needed, set up Zotero/Mendeley sync
- [ ] 13.2.5: Establish communication channel — dedicated Slack/Discord channel per lab

### Task 13.3: Active Pilot (Weeks 1–8)

- [ ] 13.3.1: Weekly check-in — 15-minute video call, capture blockers, feature requests, bugs
- [ ] 13.3.2: Usage telemetry (opt-in) — commands run, papers retrieved, hypotheses generated, time spent
- [ ] 13.3.3: Bug tracking — dedicated GitHub label for pilot-reported issues, 48-hour acknowledgment SLA
- [ ] 13.3.4: Feature baselines — record time and quality of literature review without tool vs. with tool
- [ ] 13.3.5: Milestone at Week 4: each lab should have generated at least 3 testable hypotheses

### Task 13.4: Feedback Collection

- [ ] 13.4.1: Mid-pilot survey (Week 4) — NPS score, feature satisfaction, missing features, integration needs
- [ ] 13.4.2: End-of-pilot survey (Week 8) — detailed experience report, willingness to pay, referral likelihood
- [ ] 13.4.3: Structured interview (Week 8) — recorded 30-minute session per lab
- [ ] 13.4.4: Collect specific quotes for YC application (with attribution permission)

### Task 13.5: Metrics Compilation

- [ ] 13.5.1: Aggregate metrics — total papers retrieved, hypotheses generated, hypotheses experimentally tested, validated vs. invalidated ratio
- [ ] 13.5.2: Time savings — estimated hours saved per literature review (compare to baseline)
- [ ] 13.5.3: Quality metrics — hypothesis novelty score, falsification rate, user satisfaction
- [ ] 13.5.4: Bug/issue metrics — count, severity distribution, resolution time

### Task 13.6: Case Study Production

- [ ] 13.6.1: Write 2–3 case studies (one per lab, with lab approval)
- [ ] 13.6.2: Each case study: research question → tool workflow → hypotheses generated → experimental outcome (if available)
- [ ] 13.6.3: Include direct quotes, metrics screenshots, comparative time savings
- [ ] 13.6.4: Publish case studies (with lab permission) on project website and include in YC application

---

**Phase 13 Exit Criteria**: At least 2 labs with positive outcomes and willing testimonials. Aggregate metrics package ready for YC application. Bug list triaged into roadmap priorities.
