# Phase 7 — Hypothesis Generation & Experimental Design

**Duration**: 4 weeks (Nov–Dec 2026)
**Dependencies**: Phase 5 (RAG), Phase 6 (agents)
**Output**: Evidence-grounded hypothesis generation with verifiability scoring and cost-of-test estimation

---

### Task 7.1: HypothesisGenerator

- [ ] 7.1.1: Literature-grounded hypothesis synthesis — given literature review, identify gaps and connections
- [ ] 7.1.2: Hypothesis template library — PICO (Population/Intervention/Comparison/Outcome), FINER (Feasible/Interesting/Novel/Ethical/Relevant), PO as scientific structure
- [ ] 7.1.3: Multi-hypothesis generation — propose multiple competing hypotheses for same phenomenon
- [ ] 7.1.4: Hypothesis prioritization — score by novelty, plausibility, testability, impact
- [ ] 7.1.5: Hypothesis provenance — each hypothesis tagged with literature sources that support and contradict it

### Task 7.2: SerendipityInjector

- [ ] 7.2.1: Cross-domain analogy — randomly sample concepts from unrelated fields; test for analogical mapping
- [ ] 7.2.2: Contradiction-driven exploration — when two sources disagree, generate hypotheses reconciling them
- [ ] 7.2.3: Controlled randomness parameter — user-configurable "serendipity level" (0 = conservative consensus, 1.0 = maximal exploration)
- [ ] 7.2.4: Novelty scoring — measure how far a hypothesis deviates from dominant paradigm in its field
- [ ] 7.2.5: Serendipity audit trail — log all random/analogical connections for reproducibility

### Task 7.3: ExperimentalDesignAdvisor

- [ ] 7.3.1: Experiment type classification — computational experiment, wet lab, field study, clinical trial, meta-analysis
- [ ] 7.3.2: Control variable identification — suggest controls, blinding strategies, randomization
- [ ] 7.3.3: Sample size estimation — power analysis for statistical significance
- [ ] 7.3.4: Statistical method suggestion — appropriate tests for data type, distribution, sample size
- [ ] 7.3.5: Protocol template generation — structured experimental protocol with step-by-step instructions
- [ ] 7.3.6: Reagent/resource checklist — from protocol, identify required materials and equipment
- [ ] 7.3.7: Synthetic dataset generation — for power analysis, create synthetic test data with watermarking

### Task 7.4: VerifiabilityAssessor (with Cost-of-Test)

- [ ] 7.4.1: Falsifiability scoring — Popperian criterion: is there a conceivable observation that would disprove this hypothesis?
- [ ] 7.4.2: Testability classification — immediately testable (≤2 weeks, ≤$1K), testable (<6 months, <$100K), expensive (can be tested but costly), currently infeasible (technology doesn't exist, unaffordable)
- [ ] 7.4.3: Cost-of-test estimation:
  - Time: estimated person-hours and calendar time
  - Money: reagent/equipment/compute costs
  - Ethics: IRB requirements, animal/human subject considerations, dual-use risk
  - Access: specialized equipment availability, expertise availability
- [ ] 7.4.4: Practical verifiability rating — composite score combining falsifiability + cost + ethics + access
- [ ] 7.4.5: Verification path suggestion — concrete next steps to attempt verification

### Task 7.5: RelatedWorkAnalyzer

- [ ] 7.5.1: Gap identification — systematic literature gap analysis (what hasn't been studied)
- [ ] 7.5.2: Contradiction clustering — group conflicting findings and show supporting evidence for each side
- [ ] 7.5.3: Unexplored connection discovery — find papers that are rarely co-cited but address related concepts
- [ ] 7.5.4: Citation trajectory analysis — rising, peaking, declining, or dormant research areas
- [ ] 7.5.5: Research frontier mapping — identify most active and most neglected sub-areas

### Task 7.6: Synthetic Data Watermarking

- [ ] 7.6.1: Cryptographic watermarking for all generated synthetic datasets — hash-based (SHA-256 watermark embedded in metadata)
- [ ] 7.6.2: Metadata marker — every synthetic dataset includes `synthetic_source: skepsis_generated` in provenance
- [ ] 7.6.3: Watermark verification — check if a dataset was generated by Skepsis
- [ ] 7.6.4: Anti-leak warning — when exporting synthetic data, warn user that it must not be presented as empirical

### Task 7.7: Hypothesis Tests

- [ ] 7.7.1: Unit tests for HypothesisGenerator — template application, multi-hypothesis generation, prioritization
- [ ] 7.7.2: Unit tests for SerendipityInjector — cross-domain analogy, novelty scoring
- [ ] 7.7.3: Unit tests for ExperimentalDesignAdvisor — experiment classification, control identification, statistical method suggestion
- [ ] 7.7.4: Unit tests for VerifiabilityAssessor — falsifiability scoring, cost estimation, practical verifiability rating
- [ ] 7.7.5: Unit tests for RelatedWorkAnalyzer — gap identification, contradiction clustering
- [ ] 7.7.6: Integration test: literature review → gap identification → hypothesis generation → serendipity injection → experimental design → verifiability assessment → watermarking

---

**Phase 7 Exit Criteria**: Can take a literature review, generate multiple hypotheses with serendipity, design an experiment to test them, estimate cost and verifiability, and watermark any synthetic data generated.
